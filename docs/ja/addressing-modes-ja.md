# アドレッシングモード

> 根拠：一次資料 p.65–p.72

## データサイズ

SC62015 は 1・2・3 バイトのデータを扱えます。
レジスタに対しては 1（A/IL）・2（BA/I）・2.5バイト（20ビット、X/Y/U/S）です。

内部 RAM または外部メモリ空間だけをアクセスする場合（レジスタを使わない場合）は、
扱いたいサイズをニモニックで明示します。

| ニモニック suffix | サイズ |
|-----------------|--------|
| MV（サフィックスなし） | 1バイト |
| MVW | 2バイト |
| MVP | 3バイト |
| MVL | I レジスタのバイト数分（ループ） |

## 内部 RAM のアドレス指定

SC62015 は内部 RAM を 256 バイト持ちます。
オペランドが `(n)` と表記されているとき、以下の 4 種類の実効アドレスが選べます。

| 種類 | 実効アドレス | 説明 |
|------|-------------|------|
| ❶ (n)      | n 番地（絶対） | 内部 RAM の n 番地を直接指定 |
| ❷ (BP+n)   | BP + n 番地 | ベースポインタからの相対 |
| ❸ (PX+n) / (PY+n) | PX（またはPY）+ n | PX/PY ポインタからの相対 |
| ❹ (BP+PX) / (BP+PY) | BP + PX（またはPY） | ベースポインタ + ポインタ |

- **BP**（ベースポインタ）：内部 RAM の ECh 番地
- **PX**（PX ポインタ）：内部 RAM の EDh 番地
- **PY**（PY ポインタ）：内部 RAM の EEh 番地

種類の選択は命令直前の **プレバイト** で行います（`prebyte-ja.md` 参照）。

## 外部メモリ空間のアドレッシング

外部メモリ空間へのアドレスは `[` と `]` で囲んで表記します。
7 種類のアドレッシングモードがあります。

| 記法 | 説明 |
|------|------|
| `[lmn]`   | 20ビット絶対アドレス指定（l は上位4ビット） |
| `[r3]`    | r3 レジスタの値を実効アドレスとする |
| `[r3+n]`  | r3 + n を実効アドレスとする |
| `[r3++]`  | r3 の値で実行後、r3 += データサイズ |
| `[--r3]`  | r3 -= データサイズ後、r3 の値で実行 |
| `[(n)]`   | 内部 RAM の n 番地の3バイト値を実効アドレスとする |
| `[(m)±n]` | 内部 RAM の m 番地の3バイト値 ± n を実効アドレスとする |

r3 として使えるのは **X, Y, U, S** のみです。

## オペランド記法まとめ

| 記法 | 意味 |
|------|------|
| `n`       | 8ビット即値 |
| `mn`      | 16ビット即値（n が低バイト、m が高バイト） |
| `lmn`     | 20ビット即値（n が最低バイト、l が最高バイト） |
| `(n)`     | 内部 RAM アドレス n 番地 |
| `(m),(n)` | 両オペランドとも内部 RAM アドレス |
| `[lmn]`   | 外部メモリ絶対アドレス 20ビット |
| `[r3]`    | レジスタ間接 |
| `[(n)]`   | 内部 RAM 間接 |

## ループ命令

ニモニックの末尾が `L` の命令（MVL, ADCL, SBCL, EXL 等）はループ命令です。
I レジスタの値の回数だけ繰り返し実行します。
I=0 のとき 10000h（65536）回実行とみなします。
実行後、I レジスタは 0 になります。
