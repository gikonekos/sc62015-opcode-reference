# MV / MVW / MVP / MVL / MVLD

データ転送命令。

**出典:** *PC-E650/PC-U6000 活用研究*, 工学社, 1995, ISBN 4-87593-206-5 — pp.73–79

---

## 概要

SC62015 には以下の関連転送命令がある:

| ニーモニック | サイズ | 説明 |
|------------|--------|------|
| `MV`     | 1 バイト | 1 バイト転送 |
| `MVW`    | 2 バイト | 2 バイト（ワード）転送 |
| `MVP`    | 3 バイト | 3 バイト（ポインタ）転送 |
| `MVL`    | I×バイト | `I` レジスタを使った前方向繰り返し転送 |
| `MVLD`   | I×バイト | `I` レジスタを使った後方向繰り返し転送 |

**フラグ:** MV ファミリーのすべての命令でフラグは変化しない (p.73)。

`MV` が転送するデータサイズはレジスタまたはオペランドの種類に依存する:

| サイズ | 条件 |
|--------|------|
| 8 ビット  | r1 (A, IL) または内部 RAM 1 バイト |
| 16 ビット | r2 (BA, I) |
| 20 ビット | r3 (X, Y, U, S) |

レジスタによってサイズが決まらない内部 RAM ↔ 内部 RAM または内部 RAM ↔ 外部メモリの転送には、**MVW** (2 バイト) または **MVP** (3 バイト) を明示的に使用すること (p.73)。

---

## 表記

### レジスタクラス

| クラス | レジスタ | サイズ |
|-------|---------|--------|
| `r1`  | A, IL     | 8 ビット |
| `r2`  | BA, I     | 16 ビット |
| `r3`  | X, Y, U, S | 20 ビット |

### 内部 RAM オペランド（丸括弧）

`(n)`、`(BP+n)`、`(PX+n)`、`(PY+n)`、`(BP+PX)`、`(BP+PY)`

`(n)` の具体的な意味は **プレバイト** に依存する。`docs/prebyte.md` を参照。

### 外部メモリオペランド（角括弧）

`[lmn]`、`[r3]`、`[r3++]`、`[--r3]`、`[r3+n]`、`[(n)]`、`[(m)+n]`、`[(l)-n]`

## 1. 即値転送形式

### `MV r, n` — レジスタへの即値ロード

| ニーモニック | オペランド | オペコード | バイト数 | サイクル数 |
|------------|-----------|------------|---------|-----------|
| MV | A, n    | `08` n     | 2 | 2 |
| MV | IL, n   | `09` n     | 2 | 3 |
| MV | BA, mn  | `0A` n m   | 3 | 3 |
| MV | I, mn   | `0B` n m   | 3 | 3 |
| MV | X, lmn  | `0C` n m l | 4 | 4 |
| MV | Y, lmn  | `0D` n m l | 4 | 4 |
| MV | U, lmn  | `0E` n m l | 4 | 4 |
| MV | S, lmn  | `0F` n m l | 4 | 4 |

即値のバイト順: 低位バイトが先 (p.224)。

### 内部 RAM への即値転送

| ニーモニック | オペランド   | オペコード        | バイト数 | サイクル数 |
|------------|------------|---------------|---------|-----------|
| MV   | (m), n      | `CC` m n      | 3 | 3 |
| MVW  | (l), mn     | `CD` l m n    | 4 | 4 |
| MVP  | (k), lmn    | `DC` k n m l  | 5 | 5 |

## 2. レジスタ ↔ レジスタ

| ニーモニック | オペランド   | オペコード      | バイト数 | サイクル数 |
|------------|------------|-------------|---------|-----------|
| MV | A, B        | `74`        | 1 | 1 |
| MV | B, A        | `75`        | 1 | 1 |
| MV | r2, r2'     | `FD` r r'   | 2 | 2 |
| MV | r3, r3'     | `FD` r r'   | 2 | 2 |

`FD` 形式の第 2 バイト: 上位ニブル = 転送元レジスタ、下位ニブル = 転送先レジスタ。

## 4. 内部 RAM ↔ 内部 RAM

ダイレクト以外のアドレッシングモードにはプレバイトが必要 (p.74)。`docs/prebyte.md` を参照。

### 1・2・3 バイト転送

| ニーモニック | オペランド   | オペコード     | バイト数 | サイクル数 |
|------------|------------|------------|---------|-----------|
| MV   | (m), (n)  | `C8` m n   | 3 | 6  |
| MVW  | (m), (n)  | `C9` m n   | 3 | 8  |
| MVP  | (m), (n)  | `CA` m n   | 3 | 10 |

### 繰り返し転送

| ニーモニック | オペランド   | オペコード     | バイト数 | サイクル数   |
|------------|------------|------------|---------|----------|
| MVL  | (m), (n)  | `CB` m n   | 3 | 5+2×I |
| MVLD | (m), (n)  | `CF` m n   | 3 | 5+2×I |

MVL および MVLD 実行後は I ← 0 になる。

## 6. レジスタ ↔ 外部メモリ（レジスタ間接）

ロード時の第 1 バイト = `(90h + r)`、ストア時の第 1 バイト = `(B0h + r)`。第 2 バイトでアドレッシングモードと r3 をエンコードする。

### `MV r, [r3]` とその派生形式

| オペランド      | 第 1 バイト  | 第 2 バイト | バイト数 | サイクル数 (r1) |
|---------------|-------------|------------|---------|----------------|
| r, [r3]       | `(90h+r)`   | `0r3`      | 2 | 4 |
| r, [r3++]     | `(90h+r)`   | `2r3`      | 2 | 4 |
| r, [--r3]     | `(90h+r)`   | `3r3`      | 2 | 5 |
| r, [r3+n]     | `(90h+r)`   | `8r3` n    | 3 | 6 |
| r, [r3-n]     | `(90h+r)`   | `Cr3` n    | 3 | 6 |

### `MV [r3], r` とその派生形式

| オペランド      | 第 1 バイト  | 第 2 バイト | バイト数 | サイクル数 (r1) |
|---------------|-------------|------------|---------|----------------|
| [r3], r       | `(B0h+r)`   | `0r3`      | 2 | 4 |
| [r3++], r     | `(B0h+r)`   | `2r3`      | 2 | 4 |
| [--r3], r     | `(B0h+r)`   | `3r3`      | 2 | 5 |
| [r3+n], r     | `(B0h+r)`   | `8r3` n    | 3 | 6 |
| [r3-n], r     | `(B0h+r)`   | `Cr3` n    | 3 | 6 |

## MVW opr1, opr2

**動作:** opr1L ← opr2L, opr1H ← opr2H  
**フラグ:** 影響なし

MV の 2 バイト（16 ビット）版。連続する 2 バイトを転送する。低位バイトが低位アドレスに格納される。

オペランドの組み合わせはセクション 4・8 の形式の一部。レジスタオペランドには使用不可（レジスタのサイズはレジスタ自身が決定する）。

---

## MVP opr1, opr2

**動作:** opr1L ← opr2L, opr1M ← opr2M, opr1H ← opr2H  
**フラグ:** 影響なし

MV の 3 バイト（20 ビット）版。連続する 3 バイトを低位バイトから順に転送する。

---

## MVL opr1, opr2

**動作:** I レジスタのバイト数分、opr2 領域から opr1 領域へ転送し、両アドレスをインクリメントする。  
**フラグ:** 影響なし (p.77)  
**実行後:** I ← 0

ループ転送命令。転送を I 回繰り返し、各反復で転送元・転送先アドレスをそれぞれ 1 インクリメントする。開始時に I = 0 の場合、10000h (65536) 回実行される (p.77)。

オペランドの組み合わせ: 内部 RAM ↔ 内部 RAM、および内部 RAM ↔ 外部メモリのみ (p.77)。セクション 4・8 を参照。

**MVL 実行前に I レジスタをセットすること。** 転送元と転送先の領域が重複している場合は、未読データを上書きしないよう **MVLD** を使用すること (p.79)。

### 使用例

```
; 内部 RAM 20h–23h の 4 バイトを 10h–13h にコピー
; (絶対アドレッシング、プレバイト 32h)
MV    I, 04h      ; 0B 04 00
PRE   32h
MVL   (10h),(20h) ; CB 10 20
; 実行後: I = 0、10h–13h にコピーされたデータが入る
```

---

## MVLD opr1, opr2

**動作:** I レジスタのバイト数分、opr2 領域から opr1 領域へ転送し、両アドレスをデクリメントする。  
**フラグ:** 影響なし (p.79)  
**実行後:** I ← 0

アドレスをインクリメントではなくデクリメントする点を除けば MVL と同一 (p.79)。

オペランド: 内部 RAM ↔ 内部 RAM のみ — オペコード `CF` m n。

転送元と転送先の領域が重複しており、転送先が転送元より**高位アドレス**にある場合は、MVL の代わりに MVLD を使用すること。MVL は低位から高位へコピーするため、この状況では未読の転送元データを上書きしてしまう。MVLD は高位から低位へコピーするためこの問題を回避できる (p.79)。

### 使用例

```
; 内部 RAM 10h–1Fh を 20h–2Fh に移動
; (重複領域のため MVL ではなく MVLD を使用)
MV    I, 10h      ; I = 16
PRE   32h
MVLD  (3Fh),(2Fh) ; CF 3F 2F   (上位端から開始)
```

## 備考

- **IL への書き込みと同時に IH が 0 にクリアされる** (p.224)。IL を転送先とするすべての MV 命令に適用される。
- 内部 RAM 間接アドレッシングモードにはプレバイトが必要。`docs/prebyte.md` を参照 (p.68)。
- 一部の MV 形式は第 1 バイトが共通で、第 2 バイトで区別される。特に `90h–96h`、`98h–9Eh`、`B0h–B6h`、`B8h–BEh` の各ファミリーでは、バイナリをデコードする際に第 2 バイトを慎重に解釈すること。
- 本ドキュメントには出典ページから確認された形式のみを記載している。一見対称的に見える形式でも、出典での確認なしに存在を想定してはならない。

## 使用例

```
; MV A, B  —  B を A にコピー
74

; MV X, 0BDC00h  —  20 ビット即値を X にロード
0C 00 DC 0B

; MV A, (10h)  —  内部 RAM アドレス 10h を A にロード
80 10

; MV (20h), A  —  A を内部 RAM アドレス 20h に格納
A0 20

; MV A, [X]  —  X のアドレスにある外部メモリから A にロード  (r=0=A, r3=4=X)
90 04

; MV [X++], A  —  A を [X] に格納し、X += 1
B0 24
```
