# MV / MVW / MVP / MVL / MVLD

データ転送命令。

**出典:** *PC-E650/PC-U6000 活用研究*, 工学社, 1995, ISBN 4-87593-206-5 — pp.73–79

---

## MV opr1, opr2

**動作:** opr1 ← opr2  
**フラグ:** 影響なし (p.73)

opr2 の内容を opr1 にコピーする。転送するデータサイズはレジスタまたはオペランドの種類に依存する:

| サイズ | 条件 |
|--------|------|
| 8 ビット | r1 (A, IL) または内部 RAM 1 バイト |
| 16 ビット | r2 (BA, I) |
| 20 ビット | r3 (X, Y, U, S) |

レジスタによってサイズが決まらない内部 RAM ↔ 内部 RAM または内部 RAM ↔ 外部メモリの転送には、**MVW** (2 バイト) または **MVP** (3 バイト) を明示的に使用すること (p.73)。

### オペランドの組み合わせ

#### ❶ 即値 → レジスタ または 内部 RAM (p.73)

| opr1 | opr2 | オペコード |
|------|------|------------|
| r1 | n | `(08+r1)` n |
| r2 | mn | `(08+r2)` n m |
| r3 | lmn | `(08+r3)` n m l |
| (m) | n | `CC` m n |
| MVW (l), mn | — | `CD` l m n |
| MVP (k), lmn | — | `DC` k n m l |

#### ❷ レジスタ ↔ レジスタ (p.74)

| opr1 | opr2 | オペコード |
|------|------|------------|
| A | B | `74` |
| B | A | `75` |
| r2 | r2' | `FD` r2 r2' |
| r3 | r3' | `FD` r3 r3' |

#### ❸ 内部 RAM ↔ 内部 RAM (p.74)

プレバイトが必要。`docs/prebyte.md` を参照。

| ニーモニック | opr1, opr2 | オペコード |
|------------|-----------|------------|
| MV | (m), (n) | `C8` m n |
| MVW | (m), (n) | `C9` m n |
| MVP | (m), (n) | `CA` m n |

#### ❹ レジスタ ↔ 内部 RAM (p.74)

プレバイトを使用できる。

| opr1 | opr2 | オペコード |
|------|------|------------|
| r | (n) | `(8r)` n |
| (n) | r | `(Ar)` n |

#### ❺ レジスタ ↔ 外部メモリ (p.74)

> 注意: `MV r, [lmn]` および `MV [lmn], r` を除き、r ≠ S (p.74)

**MV r, [addr]:**

| opr2 アドレッシング | オペコード (第 2 バイト) |
|--------------------|--------------------------|
| [lmn] | `(88+r)` n m l |
| [r3] | `9r` `0r3` |
| [r3++] | `9r` `2r3` |
| [--r3] | `9r` `3r3` |
| [r3+n] | `9r` `8r3` n |
| [r3-n] | `9r` `Cr3` n |
| [(n)] | `(98+r)` `00` n |
| [(m)+n] | `(98+r)` `80` m n |
| [(l)-n] | `(98+r)` `C0` l n |

**MV [addr], r:**

| opr1 アドレッシング | オペコード (第 2 バイト) |
|--------------------|--------------------------|
| [lmn] | `(A8+r)` n m l |
| [r3] | `Br` `0r3` |
| [r3++] | `Br` `2r3` |
| [--r3] | `Br` `3r3` |
| [r3+n] | `Br` `8r3` n |
| [r3-n] | `Br` `Cr3` n |
| [(n)] | `(B8+r)` `00` n |
| [(m)+n] | `(B8+r)` `80` m n |
| [(l)-n] | `(B8+r)` `C0` l n |

#### ❻ 内部 RAM ↔ 外部メモリ (p.75)

| ニーモニック | opr1 | opr2 | オペコード |
|------------|------|------|------------|
| MV / MVW / MVP | (k) | [lmn] | `D0/D1/D2` k n m l |
| MV / MVW / MVP | [klm] | (n) | `D8/D9/DA` m l k n |
| MV / MVW / MVP | (n) | [r3] | `E0/E1/E2` `0r3` n |
| MV / MVW / MVP | (n) | [r3++] | `E8/E9/EA` `2r3` n |
| MV / MVW / MVP | (m) | [(n)] | `F0/F1/F2` `00` m n |
| MV / MVW / MVP | [(l)±m] | (n) | `F8/F9/FA` `80`/`C0` l m n |

---

### 備考

- **IL への書き込みと同時に IH が 0 にクリアされる** (p.224)。IL を転送先とするすべての MV 命令に適用される。
- 内部 RAM 間接アドレッシングモードにはプレバイトが必要。`docs/prebyte.md` を参照 (p.68)。

### 使用例

```
; MV A, B  —  B を A にコピー
74

; MV X, 0BDC00h  —  20 ビット即値を X にロード
0C 00 DC 0B

; MV A, (10h)  —  内部 RAM アドレス 10h を A にロード
80 10

; MV (20h), A  —  A を内部 RAM アドレス 20h に格納
A0 20

; MV A, [X]  —  X のアドレスにある外部メモリから A にロード  (r=0=A, r3=4=X)
90 04

; MV [X++], A  —  A を [X] に格納し、X += 1
B0 24
```

## MVW opr1, opr2

**動作:** opr1L ← opr2L, opr1H ← opr2H  
**フラグ:** 影響なし

MV の 2 バイト (16 ビット) 版。連続する 2 バイトを転送する。低位バイトが低位アドレスに格納される。

オペランドの組み合わせは MV の ❸❻ の一部 (p.73, p.75)。レジスタオペランドには使用不可（レジスタのサイズはレジスタ自身が決定する）。

---

## MVP opr1, opr2

**動作:** opr1L ← opr2L, opr1M ← opr2M, opr1H ← opr2H  
**フラグ:** 影響なし

MV の 3 バイト (20 ビット) 版。連続する 3 バイトを低位バイトから順に転送する。

---

## MVL opr1, opr2

**動作:** I レジスタのバイト数分、opr2 領域から opr1 領域へ転送し、両アドレスをインクリメントする。  
**フラグ:** 影響なし (p.77)  
**実行後:** I ← 0

ループ転送命令。I レジスタの回数だけ MV を実行し、転送のたびに転送元・転送先アドレスをそれぞれ 1 インクリメントする。I = 0 のときは 10000h (65536) 回実行される (p.77)。

オペランドの組み合わせ: ❸ 内部 RAM ↔ 内部 RAM、および ❻ 内部 RAM ↔ 外部メモリのみ (p.77)。

> 注意: `MVL (n), [r3]` および `MVL [r3], (n)` は存在しない (p.77)。

### 備考

- **MVL 実行前に I レジスタをセットすること。** 転送するバイト数を事前に I にロードしておく必要がある。
- **実行後、I = 0 になる。**
- 転送元と転送先が重複している場合は、未読データを上書きしないよう **MVLD** を使用すること (p.79)。

### 使用例

```
; 内部 RAM 20h–23h の 4 バイトを 10h–13h にコピー
; (絶対アドレッシング、プレバイト 32h)

MV    I, 04h     ; I = 4  → 0B 04 00
PRE   32h        ; (m)×(n) プレバイト
MVL   (10h),(20h) ; CB 10 20
; 実行後: I = 0、10h–13h にコピーされたデータが入る
```

## MVLD opr1, opr2

**動作:** I レジスタのバイト数分、opr2 領域から opr1 領域へ転送し、両アドレスをデクリメントする。  
**フラグ:** 影響なし (p.79)  
**実行後:** I ← 0

アドレスをインクリメントではなくデクリメントする点を除けば MVL と同一のループ転送命令 (p.79)。オペコード: `CF` m n。

オペランド: 内部 RAM ↔ 内部 RAM のみ (`(m), (n)`)。

### MVL ではなく MVLD を使うべき場合

転送元と転送先の領域が重複しており、かつ転送先が転送元より高位アドレスにある場合は MVLD を使用すること。MVL は常に低位アドレスから高位アドレスへコピーするため、転送先が転送元の高位側に重複していると、未読データを上書きしてしまう。MVLD は高位から低位へコピーするため、この問題を回避できる (p.79)。

### 使用例

```
; 内部 RAM 10h–1Fh を 20h–2Fh に移動
; (重複領域のため MVL ではなく MVLD を使用)

MV    I, 10h      ; I = 16
PRE   32h
MVLD  (3Fh),(2Fh) ; CF 3F 2F   (上位端から開始)
```
