# コール・リターン命令

**出典:** *PC-E650/PC-U6000 活用研究*, 工学社, 1995, ISBN 4-87593-206-5 — pp.98–99, 103

---

## CALL mn — ショートコール (16 ビット)

**動作:** 次の命令アドレス (16 ビット PC) をシステムスタックにプッシュし、PC ← mn  
**フラグ:** 影響なし (p.98)

16 ビットアドレスのサブルーチンを呼び出す。PS は変化しないため、呼び出し可能範囲は現在の PS セグメント内の 64K バイトに限られる。RET で戻る (p.98)。

| opr | オペコード |
|-----|------------|
| mn | `04` n m |

**使用例:**
```
; CALL 80A2h — PS は変化せず、(PS, 80A2h) を呼び出す
04 A2 80
```

---

## RET — CALL からのリターン

**動作:** PC ← システムスタックからポップ  
**フラグ:** 影響なし (p.98)

CALL で呼び出したサブルーチンから戻る。システムスタックから 16 ビットのリターンアドレスをポップする (p.98)。

| オペコード |
|------------|
| `06` |

---

## CALLF lmn — ファーコール (20 ビット)

**動作:** 次の命令アドレス (20 ビット: PC と PS) をシステムスタックにプッシュし、PC ← lmn の下位 16 ビット、PS ← lmn の上位 4 ビット  
**フラグ:** 影響なし (p.99)

完全な 20 ビットアドレスのサブルーチンを呼び出す。PC と PS の両方が保存・更新される。アドレス空間: 1M バイト。RETF で戻る (p.99)。

| opr | オペコード |
|-----|------------|
| lmn | `05` n m l |

**使用例:**
```
; CALLF 0BDD69h — BDD69h を呼び出す
05 69 DD 0B
```

---

## RETF — CALLF からのリターン

**動作:** PC ← システムスタックからポップ、PS ← システムスタックからポップ  
**フラグ:** 影響なし (p.99)

CALLF で呼び出したサブルーチンから戻る。システムスタックから PC と PS の両方をポップする (p.99)。

| オペコード |
|------------|
| `07` |

---

## RETI — 割り込みからのリターン

**動作:** IMR ← (S), C,Z ← (S+1), PCL ← (S+2), PCH ← (S+3), PS ← (S+4), S ← S+5  
**フラグ:** スタックから復元される (p.103)

割り込みハンドラから戻る。IMR、フラグ、PC、PS をこの順でシステムスタックからポップし、RETF を実行する (p.103)。

| オペコード |
|------------|
| `01` |

### 備考

- フラグを復元するリターン命令は RETI のみ。RET と RETF はフラグに影響しない。
- IMR (割り込みマスクレジスタ) は内部 RAM アドレス FBh にある (p.103)。
