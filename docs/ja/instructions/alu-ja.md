# 算術演算命令

**出典:** *PC-E650/PC-U6000 活用研究*, 工学社, 1995, ISBN 4-87593-206-5 — pp.84–88

---

## ADD opr1, opr2

**動作:** opr1 ← opr1 + opr2  
**フラグ:** Z: 結果が 0 のときセット / C: キャリーまたはボローが発生したときセット (p.84)

| opr1 | opr2 | オペコード |
|------|------|------------|
| A | n | `40` n |
| (m) | n | `41` m n |
| A | (n) | `42` n |
| (n) | A | `43` n |
| r2, r1' | — | `44` (PARTIAL — オペコード表を参照) |
| r3 | r' | `45` r3r' |
| r1' | r1' | `46` (PARTIAL — オペコード表を参照) |

---

## SUB opr1, opr2

**動作:** opr1 ← opr1 − opr2  
**フラグ:** Z: 結果が 0 のときセット / C: キャリーまたはボローが発生したときセット (p.84)

| opr1 | opr2 | オペコード |
|------|------|------------|
| A | n | `48` n |
| (m) | n | `49` m n |
| A | (n) | `4A` n |
| (n) | A | `4B` n |
| r2 | r1' | `4C` (PARTIAL) |
| r3 | r' | `4D` r3r' |
| r1' | r1' | `4E` (PARTIAL) |

---

## ADC opr1, opr2

**動作:** opr1 ← opr1 + opr2 + C  
**フラグ:** Z: 結果が 0 のときセット / C: キャリーが発生したときセット (p.85)

キャリー付き加算。それ以外は ADD と同じ (p.85)。

| opr1 | opr2 | オペコード |
|------|------|------------|
| A | n | `50` n |
| (m) | n | `51` m n |
| A | (n) | `52` n |
| (n) | A | `53` n |

---

## SBC opr1, opr2

**動作:** opr1 ← opr1 − opr2 − C  
**フラグ:** Z: 結果が 0 のときセット / C: ボローが発生したときセット (p.85)

キャリー付き減算。それ以外は SUB と同じ (p.85)。

| opr1 | opr2 | オペコード |
|------|------|------------|
| A | n | `58` n |
| (m) | n | `59` m n |
| A | (n) | `5A` n |
| (n) | A | `5B` n |

## ADCL opr1, opr2 — キャリー付きループ加算

**動作:** I レジスタのバイト数分、opr2 領域から opr1 領域へキャリーを伝播しながら加算し、両アドレスをインクリメントする (p.86)  
**フラグ:** Z: 結果が 0 のときセット / C: キャリーが発生したときセット (p.86)  
**実行後:** I ← 0

ループ加算命令。ADC を I 回繰り返し、バイト間でキャリーを伝播する。実行前に I にバイト数をセットしておくこと (p.86)。

| opr1 | opr2 | オペコード |
|------|------|------------|
| (m) | (n) | `54` m n |
| (n) | A | `55` n |

**使用例:**
```
; ADCL (30h), (50h)  —  30h と 50h にある 4 バイト値を加算
; (内部 RAM 絶対アドレッシング、プレバイト 32h)
; I = 0004h, C = ?
; 実行後: I = 0, オーバーフロー時は C = 1
0B 04 00    ; MV I, 4
20          ; PRE 32h (絶対アドレッシング用プレバイト)
54 30 50    ; ADCL (30h),(50h)
```

## SBCL opr1, opr2 — キャリー付きループ減算

**動作:** I レジスタのバイト数分、opr2 を opr1 からボローを伝播しながら減算し、両アドレスをインクリメントする (p.86)  
**フラグ:** Z: 結果が 0 のときセット / C: ボローが発生したときセット (p.86)  
**実行後:** I ← 0

| opr1 | opr2 | オペコード |
|------|------|------------|
| (m) | (n) | `5C` m n |
| (n) | A | `5D` n |

---

## DADL opr1, opr2 — BCD ループ加算

**動作:** I レジスタのバイト数分、opr2 から opr1 へ BCD (2 進化 10 進数) 形式で加算し、両アドレスをデクリメントする (p.87)  
**フラグ:** Z: 結果が 0 のときセット / C: キャリーが発生したときセット (p.87)  
**実行後:** I ← 0

BCD ループ加算。各バイトは 2 桁の BCD (4 ビット × 2、0〜9) を表す。ADCL と異なり、アドレスはデクリメントされる (p.87)。

| opr1 | opr2 | オペコード |
|------|------|------------|
| (m) | (n) | `C4` m n |
| (n) | A | `C5` n |

**使用例:**
```
; DADL (10h), (20h)  —  BCD 値の加算 (プレバイト 32h)
; I = 0003h (3 バイト = 6 BCD 桁)
; 109524 + 032711 = 142235 (BCD)
C4 10 20
```

---

## DSBL opr1, opr2 — BCD ループ減算

**動作:** I レジスタのバイト数分、opr2 を opr1 から BCD 形式でボローを伝播しながら減算し、両アドレスをデクリメントする (p.87)  
**フラグ:** Z: 結果が 0 のときセット / C: ボローが発生したときセット (p.87)  
**実行後:** I ← 0

| opr1 | opr2 | オペコード |
|------|------|------------|
| (m) | (n) | `D4` m n |
| (n) | A | `D5` n |

---

## INC opr — インクリメント

**動作:** opr ← opr + 1  
**フラグ:** Z: 結果が 0 のときセット / C: 変化なし (p.88)

| opr | オペコード |
|-----|------------|
| r | `6C` `0r` (第 2 バイトでレジスタを選択) |
| (m) | `6D` m |

**使用例:**
```
; INC I  —  r=3=I、第 2 バイト = 03h
6C 03
```

---

## DEC opr — デクリメント

**動作:** opr ← opr − 1  
**フラグ:** Z: 結果が 0 のときセット / C: 変化なし (p.88)

| opr | オペコード |
|-----|------------|
| r | `7C` `0r` (第 2 バイトでレジスタを選択) |
| (m) | `7D` m |

---

## PMDF opr1, opr2 — ポインタ変更

**動作:** opr1 ← opr1 + opr2。フラグ変化なし (p.88)  
**フラグ:** 影響なし (p.88)

フラグに影響しない点を除けば ADD と同一。フラグを乱さずにアドレスポインタを変更するために使用する (p.88)。

| opr1 | opr2 | オペコード |
|------|------|------------|
| (m) | n | `47` m n |
| (n) | A | `57` n |

**使用例:**
```
; PMDF (30h), A  —  A=4Ch, (30h)=21h → (30h)=6Dh, フラグ変化なし
; (内部 RAM 絶対アドレッシング、プレバイト 30h)
57 30
```
