# Methodology

## 目的

このプロジェクトの目的は、SC62015 命令セットについて、整理された検証可能なリファレンスを構築することです。

## 基本ルール

1. 可能な限り一次印刷資料の語句と表記に従う。
2. 資料にない略記や独自表現を作らない。
3. 自信をもって読めない値や表現は未解決として扱う。
4. 以下を明確に区別する。
   - 直接確認できた読取り
   - 照合に基づく再構築
   - 未解決項目

## リポジトリ方針

このリポジトリでは、不安定な途中断片よりも、最終的に見直し可能な成果物を優先します。

一時的なスキャン確認条件に依存した中間メモは、再現性が低い場合、公開ツリーから除外することがあります。

## 検証原則

オペコード表は、該当項目が利用可能な資料ページと照合できた段階で公開します。

## 言語方針

- 公開用の主ファイルは英語版 `README.md` と `docs/*.md`
- 必要に応じて日本語版も併記します
