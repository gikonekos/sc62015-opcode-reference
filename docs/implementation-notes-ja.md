# 実装上の注意事項

> 根拠：一次資料 p.65–p.103、p.224–p.231

## 逆アセンブラ実装時の注意

### 第2バイト依存命令

以下の OPコードは第1バイトが同一で、**第2バイトによってアドレッシングモードが異なります**。
逆アセンブラでは第2バイトを読んでから命令を決定する必要があります。

| 第1バイト範囲 | 命令族 | 第2バイトの意味 |
|---|---|---|
| 90h–97h | MV r,[外部メモリ] | 0r3=[r3], 2r3=[r3++], 3r3=[--r3], 8r3=[r3+n], Cr3=[r3-n] |
| 98h–9Fh | MV r,[(内部RAM間接)] | 00h=[(n)], 80h=[(m)+n], C0h=[(l)-n] |
| B0h–B7h | MV [外部メモリ],r | 0r3=[r3], 2r3=[r3++], 3r3=[--r3], 8r3=[r3+n], Cr3=[r3-n] |
| B8h–BFh | MV [(内部RAM間接)],r | 00h=[(n)], 80h=[(m)+n], C0h=[(l)-n] |
| 6Ch | INC r | 第2バイト=0000 0r でレジスタ選択 |
| 7Ch | DEC r | 第2バイト=0000 0r でレジスタ選択 |
| EDh | EX r,r' | 第2バイト=r r'（各4ビット）でレジスタ選択 |
| FDh | MV r,r' | 第2バイト=r r'（各4ビット）でレジスタ選択 |

### プレバイトの識別

OPコード 20h–27h および 30h–37h はプレバイトです。
プレバイトを受け取ったら直後の命令のオペランドのアドレス指定モードを変更します。

### bytes カウント

- OPコード表の bytes は **プレバイトを含みません**
- プレバイトがある場合は実際の命令長 = bytes + 1
- [r3+n] 形式（第2バイト 8r3/Cr3）は第2バイトに加えて n バイト追加

## IL 書き込み時の副作用

IL（I レジスタの低バイト）に書き込む命令では **IH←0 が同時実行**されます。
MV IL,xxx / POPU IL 等すべてに適用されます。

## I レジスタの値が 0 の場合

ループ命令（MVL, ADCL, EXL 等）で I レジスタが 0 のとき、
**10000h（65536）回** 実行とみなします。

## r3 の有効範囲制限

外部メモリアドレッシングで r3 を使う命令の一部に
**r≠S（Sレジスタは使用不可）** の制約があります。

- `MV r,[lmn]` 以外の [lmn] 系：r に S は指定不可
- `MV [lmn],r` 以外の [lmn] 系：r に S は指定不可

OPコード上は r=7（S）のエンコードが存在しますが、動作は保証されません。

## PUSHS / POPS について

PUSHS と POPS は **独立した OPコードを持つのは F（フラグ）のみ**です。

- PUSHS F = 4Fh（bytes:1, cycles:3）
- POPS F  = 5Fh（bytes:1, cycles:2）

その他のオペランド（r1, r2, X, Y, IMR）については、
対応する MV 命令（`MV [--S],r` 等）と同じ OPコードになります。

## DEh（HALT）の根拠

p.230 の命令表に「CE」と読める OPコードで HALT が記載されている箇所がありますが、
p.102 の命令説明ページに「HALT → DE」と明記されており、
p.231 の H=D, L=E セルにも「HALT」が確認できます。
本資料では **DEh = HALT** としています。

## SC / RC の OPコード

- SC（キャリーフラグセット）= **97h**（p.229「SC → 1001 0111」）
- RC（キャリーフラグリセット）= **9Fh**（p.229「RC → 1001 1111」）
