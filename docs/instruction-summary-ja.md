# 命令概要

この文書は、**SC62015** 命令体系の主要な命令群を簡潔に整理した概要です。

これは全体像をつかむための案内用文書であり、詳細な OPコード表そのものを置き換えるものではありません。

## 目的

SC62015 の命令体系には、主に次のような種類があります。

- データ転送・交換
- スタック操作
- 算術・論理演算
- ローテイト・シフト
- 条件分岐・無条件分岐
- コール・リターン
- その他の制御命令

この文書は、詳細表を見る前に命令体系全体の構造を把握しやすくすることを目的としています。

## データ転送・交換命令

主なニモニック:

- `MV`
- `MVW`
- `MVP`
- `MVL`
- `MVLD`
- `EX`
- `EXW`
- `EXP`
- `EXL`

これらの命令は、主に次の間でデータを転送または交換します。

- レジスタ
- 内部RAM
- 外部メモリ空間

SC62015 では、転送サイズがニモニック系列によって区別されます。
たとえば `MV`、`MVW`、`MVP` は近い関係にありますが、扱うデータサイズが異なります。

また、`MVL` や `MVLD` のようなループ転送命令は、この CPU 系の特徴のひとつです。

## スタック命令

主なニモニック:

- `PUSHS`
- `POPS`
- `PUSHU`
- `POPU`

原資料では、次の2種類が区別されています。

- **システムスタック**
- **ユーザースタック**

これらの命令は、`S` または `U` が指すスタックを使ってレジスタ内容を退避・復帰します。

## 算術命令

主なニモニック:

- `ADD`
- `SUB`
- `ADC`
- `SBC`
- `ADCL`
- `SBCL`
- `DADL`
- `DSBL`
- `INC`
- `DEC`
- `PMDF`

この分類には、次のようなものが含まれます。

- 通常の加算・減算
- キャリー付き演算
- 内部RAM上での多バイト演算
- 特定系列での10進系多バイト演算
- インクリメント・デクリメント

命令によっては、使用できるオペランド形式が限定されています。
その点は詳細表で確認する必要があります。

## 比較・論理命令

主なニモニック:

- `CMP`
- `CMPW`
- `CMPP`
- `AND`
- `OR`
- `XOR`
- `TEST`

これらは主に次の用途で使われます。

- 分岐前の比較
- ビット単位論理演算
- フラグだけを更新する判定

特に `TEST` は、演算結果を保持せずフラグだけに影響する点が重要です。

## ローテイト・シフト命令

主なニモニック:

- `ROR`
- `ROL`
- `SHR`
- `SHL`
- `DSRL`
- `DSLL`

この分類には次のものが含まれます。

- 1ビット回転
- 1ビットシフト
- 内部RAMループ形式での10進桁シフト

原資料でも、ローテイト・シフトは独立した概念群として扱われています。

## 相対ジャンプ命令

主なニモニック:

- `JR`
- `JRZ`
- `JRNZ`
- `JRC`
- `JRNC`

これらは相対分岐命令です。
現在の実行位置に対して符号付きオフセットを加える形で分岐します。

近距離の制御フローを短いコードで書くのに向いています。

## ショート・ジャンプ命令（16ビット）

主なニモニック:

- `JP`
- `JPZ`
- `JPNZ`
- `JPC`
- `JPNC`

これらは 16ビット系のジャンプ命令です。
相対ジャンプとは別系統として扱う必要があります。

## ファー・ジャンプ命令（20ビット）

主なニモニック:

- `JPF`
- `JP`

原資料では、`JP` はオペランド形式により 16ビット版と 20ビット版の両方に現れます。
この点は重要であり、再構築でもその区別を保持する必要があります。

## コール・リターン命令

主なニモニック:

- `CALL`
- `RET`
- `CALLF`
- `RETF`
- `RETI`

これらは次の処理を扱います。

- ショートコール / リターン
- ファーコール / リターン
- 割り込み復帰

`RETI` は通常の `RET` や `RETF` と同一視せず、独立した命令として扱うべきです。

## その他の命令

主なニモニック:

- `SWAP A`
- `SC`
- `RC`
- `NOP`
- `WAIT`
- `HALT`
- `OFF`
- `IR`
- `RESET`

この分類には次のようなものが含まれます。

- フラグ制御
- NOP や WAIT などの実行制御
- クロック停止命令
- ソフトウェア割り込み発生
- ソフトウェアリセット

これらの一部は単純な1バイト制御命令で、OPコード表でも重要な位置を占めます。
