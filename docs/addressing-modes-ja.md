# Addressing Modes

この文書では、再構築した **SC62015** 命令説明で使うアドレッシング表記を整理します。

## 基本区別

原資料では、次の2種類が明確に区別されています。

- **内部RAMオペランド**
- **外部メモリ空間オペランド**

表記も異なるため、混同しないことが重要です。

## 内部RAM表記

内部RAMオペランドは丸括弧で表されます。

代表例:

- `(n)`
- `(BP+n)`
- `(PX+n)`
- `(PY+n)`
- `(BP+PX)`
- `(BP+PY)`

これらがどの意味になるかは、命令直前に置かれるプレバイトに依存します。

## プレバイトと内部RAMアドレッシング

内部RAMアクセスでは、**プレバイト** が必要になる場合があります。

プレバイトは `(n)` の意味を決めます。たとえば、

- 内部RAM絶対番地
- BP相対
- PX/PY相対
- ベース＋ポインタ合成

などを指定します。

SC62015 ではこの仕組み自体が重要なので、原資料表記をそのまま保つ方針です。

## 外部メモリ空間表記

外部メモリ空間オペランドは角括弧で表されます。

代表例:

- `[lmn]`
- `[r3]`
- `[r3+n]`
- `[r3++]`
- `[--r3]`
- `[(n)]`
- `[(m)±n]`

これらはそれぞれ別の外部メモリアドレッシング方式です。

## 主な外部メモリ形式

## 1. 絶対アドレス指定

- `[lmn]`

20ビットの番地を直接指定します。

## 2. レジスタ間接

- `[r3]`

`r3` レジスタの内容を実効アドレスとして使います。

## 3. オフセット付きレジスタ間接

- `[r3+n]`
- `[r3-n]`

`r3` の値にオフセットを加減したものを実効アドレスとして使います。

## 4. 後置インクリメント

- `[r3++]`

現在の `r3` が示す番地でアクセスした後、命令が扱ったデータサイズ分だけレジスタを増加させます。

## 5. 前置デクリメント

- `[--r3]`

先にレジスタを命令のデータサイズ分だけ減少させ、その後の値を実効アドレスとして使います。

## 6. 内部RAM間接

- `[(n)]`

内部RAM上の内容から20ビットの番地を作り、それを外部メモリ空間の番地として使います。

## 7. オフセット付き内部RAM間接

- `[(m)+n]`
- `[(m)-n]`

内部RAM上の内容から作った20ビット番地に、さらにオフセットを加減して使います。

## データサイズとニモニック

SC62015 では、扱うデータサイズはオペランドだけでなくニモニック系列でも区別されます。

例:

- `MV`   : 1バイト転送
- `MVW`  : 2バイト転送
- `MVP`  : 3バイト転送
- `MVL`  : ループ転送
- `MVLD` : 逆方向ループ転送

これは、両オペランドがメモリ系表記で、レジスタからサイズが自明でない場合に特に重要です。

## 編集方針

この文書は原資料に近い表記を維持します。

したがって、

- `(n)` を現代風の擬似構文に直す
- `[r3++]` を別アセンブラ流儀に書き換える
- 内部RAMと外部メモリ空間を独自統一記法にまとめる

といったことは行いません。

## 注意

見た目が似ていても、別系統の形式として扱う必要があります。

例:

- `(n)` と `[(n)]`
- `[r3]` と `[r3+n]`
- `[r3++]` と `[--r3]`

は、それぞれ別物です。
