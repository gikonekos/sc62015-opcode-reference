# プレバイト

> 根拠：一次資料 p.67–p.68

## プレバイトとは

プレバイトは、命令の直前に置く 1 バイトの補助コードです。
オペランドの `(n)` が指す内部 RAM のアドレス指定モードを切り替えます。

プレバイトがない場合は **絶対アドレス指定（❶）** になります。

## プレバイト表

2つのオペランドそれぞれのアドレス指定モードの組み合わせで
プレバイトの値が決まります。

| 1つ目のオペランド | 2つ目のオペランド | プレバイト |
|---|---|---|
| (n) | (n) | 32h |
| (n) | (BP+n) | 30h |
| (n) | (PY+n) | 33h |
| (n) | (BP+PY) | 31h |
| (BP+n) | (n) | 22h |
| (BP+n) | (BP+n) | なし |
| (BP+n) | (PY+n) | 23h |
| (BP+n) | (BP+PY) | 21h |
| (PX+n) | (n) | 36h |
| (PX+n) | (BP+n) | 34h |
| (PX+n) | (PY+n) | 37h |
| (PX+n) | (BP+PY) | 35h |
| (BP+PX) | (n) | 26h |
| (BP+PX) | (BP+n) | 24h |
| (BP+PX) | (PY+n) | 27h |
| (BP+PX) | (BP+PY) | 25h |

## 読み方

- 1つ目のオペランド: 第1オペランド（または唯一のオペランド）のアドレス指定モード
- 2つ目のオペランド: 第2オペランドのアドレス指定モード
- プレバイト: その組み合わせに対応する16進プレバイト値

**例:**

    27 CB 00 0A    MVL (BP+PX),(PY+0Ah)
    ^              プレバイト 27h = (PX+m) × (PY+n) の組み合わせ

## OPコードとの対応

プレバイトは OPコード体系上 20h–27h および 30h–37h に割り当てられています。

| プレバイト | opr1 | opr2 |
|---|---|---|
| 20h | (n) | (BP+n) |
| 21h | (n) | (BP+PY) |
| 22h | (BP+n) | (n) |
| 23h | (BP+n) | (PY+n) |
| 24h | (BP+PX) | (BP+n) |
| 25h | (BP+PX) | (BP+PY) |
| 26h | (BP+PX) | (n) |
| 27h | (BP+PX) | (PY+n) |
| 30h | (m) | (BP+n) |
| 31h | (m) | (BP+PY) |
| 32h | (m) | (n) |
| 33h | (m) | (PY+n) |
| 34h | (PX+m) | (BP+n) |
| 35h | (PX+m) | (BP+PY) |
| 36h | (PX+m) | (n) |
| 37h | (PX+m) | (PY+n) |

## 特記事項

- プレバイトの bytes / cycles は直後の命令に加算される形になります
- **❹（BP+PX または BP+PY）** の場合、命令実行上 `n` の指定は不要ですが、OPコード作成時には `00h` などのダミー値を置く必要があります
- **(BP+n) × (BP+n)** の組み合わせにはプレバイトがありません
- OPコード表上の `bytes` はプレバイトを含みません
